14 · Implementation & observability

Ten dna-* services, fifteen SLOs, one distributed trace

The SOTA methods are nothing without the plumbing. Ten new services on GKE, a unified event bus with versioned protobuf contracts, and OpenTelemetry traces that stitch one training cycle from Kalodata ingest through judge → OPE → bandit → rollout → DPO → ledger. Everything runs in dedicated dna-compute / dna-inference / dna-data namespaces — the baseline ScriptSage / queue-app / vidgen-engine deployment is untouched.

Go

dna-ingest

GKE dna-data · KEDA on Kafka lag

Pulls Kalodata top-seller feed, validates schema, writes to dna_training_corpus, emits dna.variant.ingested events. Owns P1.

Python

dna-judge-ensemble

GKE dna-compute · GPU A100 · HPA on queue depth

VideoReward + VisionReward + 7B distilled judge. Returns 5-dim reward vector. Owns P2 scoring.

Python

GKE dna-inference · CPU-heavy · batch every 6h

Runs β*-IPS + DoubleML over logged bandit data. Gates every promotion candidate before any impressions are spent.

Go

dna-bandit-router

GKE dna-inference · low-latency · p99 <100ms

CMAB-MT + MOL-TS arm selection. Serves the live A/B traffic split. Reads hierarchical posteriors from ebmTS.

Python

dna-dpo-trainer

GKE dna-compute · H100 · weekly jobs

VideoDPO + ORPO fine-tunes. Reads preference pairs from the compounding ledger, writes new checkpoints to MLflow.

Python

dna-gad-adversary

GKE dna-compute · H100 · Phase 13 only

Discriminator + generator adversarial loop. Convergence criterion: discriminator AUC on (top-seller, generated) ≈ 0.5.

Python

dna-rlaif-annotator

GKE dna-inference · API-bound

Constitutional cross-check every preference pair vs brand-safety + TikTok Shop compliance rules before ledger append.

Go

dna-rollout-controller

GKE dna-inference · always-on

72-hour canary windows with continuous mSPRT monitoring. Auto-rollback on any guardrail trip.

Python

dna-drift-monitor

GKE dna-inference · scheduled 15min

PSI + KS + MMD on feature/reward distributions. Triggers retrain and alerts when drift crosses thresholds.

TS

dna-registry-gateway

GKE dna-inference · edge-facing

Single API gateway for all /api/v1/dna/* routes. mTLS between internal services, OAuth2 for brand-facing endpoints.

16 · Failure modes & safety rails

Seven failure classes, twelve circuit breakers, three kill switches

An autonomous pipeline that can spend a brand's budget without human approval must assume every component will fail in an adversarial direction. Goodhart's law, reward hacking, mode collapse, data gaming, drift, compliance drift, cascade failures — we catalog 30 failure modes across 7 classes, define formulas for each detection signal, and wire autonomous recoveries. Humans only at severity S1 or repeated S2.

F1

Reward hacking

Goodhart's law: the judge becomes the objective.

  • Judge/GMV divergence: |Rjudge − RGMV| > 2σ over 7-day window
  • Sycophantic judge: corr(judge_score, brand_prior) > 0.8
  • Spec gaming: reward ↑ while GMV-attribution ↓
  • Adversarial prompt exploits: monthly red-team tournament
  • Remedy: ORPO χ² regularization + RLAIF cross-check + shadow reward model
F2

Distribution shift

Seasonal, category-migration, concept drift.

  • PSI > 0.25 warn, > 0.5 alert
  • KS test on reward dist: p < 0.01 over 24h
  • MMD on embeddings: > threshold per-category
  • Conformal p-values trending low
  • Remedy: auto-retrain hyperpriors + canary freeze + escalate if persistent
F3

Model collapse

Generator or discriminator degenerates.

  • Effective arm count: 1/Σpi² < 3 for 72h
  • Discriminator trivial: AUC → 1.0 (generator too weak)
  • DPO overfit: train↔eval KL > 0.5
  • Catastrophic forgetting: baseline regression on held-out set
  • Remedy: DPP diversity penalty + ref-policy rollback + 10% replay buffer
F4

Data poisoning / gaming

Adversarial creators attack Kalodata feed.

  • GMV velocity spike 10× baseline, no paid push
  • Unverified seller (age <180d, return-rate ≥5%)
  • Kalodata↔Fastmoss disagreement on same video
  • Click-farm fingerprint in watch-time distribution
  • Remedy: multi-source cross-check + velocity filter + hard-negative mining
F5

Cold-start catastrophe

New brands/categories misrouted.

  • Flat Beta(1,1) prior detected on any arm
  • Brand has <5 similar neighbors in prior index
  • L3 category new & unsupervised
  • Remedy: fallback chain brand→nearest-neighbor→L3→L2→L1→platform; never flat
F6

Infra / pipeline

Backpressure, staleness, DDoS, cascades.

  • Kafka consumer lag > 10 min
  • OPE estimate staleness > 6h
  • Neon replica lag > 90s
  • Rate-limit cascade signal across >2 services
  • Remedy: circuit breakers + KEDA scale-out + graceful degradation to baseline
F7

Governance / compliance

FTC, TikTok policy, brand safety, PII.

  • FTC disclosure missing in any promoted variant
  • TikTok policy update → constitution diff > 0
  • Brand-safety classifier flag
  • PII/copyright fingerprint in prompt or output
  • Remedy: constitutional AI gate + human-in-loop on any red-flag + instant rollback

17 · Ecosystem integration

One shared learning ledger across DNA, APO, BEAST, PA, SID

DNA does not live alone. Seven MomentIQ systems exchange contracts via a versioned event bus — Kalodata, ScriptSage, SID, BEAST, PA pipeline, APO, Partner Center — and all of them write to a single learning_ledger table partitioned by month. Cross-system safety rules

ScriptSage down: DNA queues generation requests up to 6h, then graceful degrade to baseline configs.
Kalodata stale > 24h: DNA freezes new-brand cold-starts. Existing arms keep running on prior posteriors.
APO ledger writes stop: DNA continues but flags "unilateral compounding — divergence risk" in dashboards.
BEAST campaign-state lag: bandit uses last-known bid envelope, refuses to promote until refreshed.
Partner Center backfill: OPE window extends proportionally, no blind promotions.
Schema breaking change: CI (`buf breaking`) blocks PR; v2 side-by-side for 90d before deprecation.
