Who owns what — roles, on-call, and change control

The DNA pipeline is a multi-team surface. Clear ownership prevents silent drift and ensures every config change is reviewed by the right domain expert.

DRI · DNA Pipeline
Content Platform lead
  • Owns the closed-loop architecture end-to-end
  • Signs off on every new DNA config
  • Reviews weekly drift report + convergence deltas
Data owner · Kalodata source
Data intelligence lead
  • Owns export scripts + yt-dlp cookie rotation
  • Guarantees n ≥ 20 per category per run
  • On-call for Phase 1 cron failures
Model owner · Gemini extractor
AI infra lead
  • Owns prompt + 42-attribute JSON schema
  • Version-pins extractor model, cuts releases
  • Maintains label golden set (50 human-labeled videos)
Generation owner · ScriptSage + vidgen
Generation lead
  • Owns hook_selector.ts, cta_builder.ts, and siblings
  • Guarantees dna_vertex / dna_pollo / dna_seedance configs remain additive
  • On-call for Phase 3–4 generation drift
Feedback owner · TikTok ingest
Partner center lead
  • Owns TikTok Business API client + 6h cron
  • Guarantees feedback idempotency
  • First responder for feedback gaps > 12h
Learning owner · Bandits + drift
ML engineering lead
  • Owns Thompson posterior code + JS-divergence detector
  • Signs off on every bandit reset
  • Maintains the learning ledger (append-only)

Change-control rules

  • DNA config edits require DRI + Generation owner approval. A CI check blocks any PR that touches the vertex_standard, pollo_standard, or seedance_standard files.
  • Rubric threshold changes (composite ≥ 0.70, voice ≥ 0.88) require DRI approval and a shadow-evaluation report attached to the PR.
  • Bandit reset (manual) requires Learning owner approval and Slack announcement to #dna-alerts. Automatic resets (3-week drift rule) post to Slack automatically.
  • New category onboarding is automatic above n = 50; below that floor, Learning owner manually blesses blended-prior sources.
  • API surface — any new /api/v1/dna/* endpoint requires DRI review, OpenAPI spec, and a companion entry in docs/dna-api-reference.md.
