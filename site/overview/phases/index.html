Kalodata Export

source
  • Login at kalodata.com (enterprise browser account)
  • Filter: US · Last 30 days · sort by Revenue desc · min GMV $10,000 · limit 200
  • Export fields: video_url, revenue_usd, orders, views, gpm, category_l1/l2, seller_id, video_created_at
  • Category-specific runs apply category_l1 filter with ≥50 videos minimum
  • Refresh listing between exports — the export dropdown breaks after first download

02

Video Download

processing
  • yt-dlp with --cookies-from-browser chrome for authenticated access
  • 4 parallel workers (TikTok rate-limit safe) via ThreadPoolExecutor
  • Format: bestvideo[height<=720]+bestaudio, merged to mp4
  • Expected failure rate 5–12% (deleted, geo-blocked, private) — do not retry >1x
  • Archive run folder to S3 after analysis; ~15 MB × 200 ≈ 3 GB

Gemini Vision Analysis

processing
  • Model: gemini-2.0-flash (~$0.05/video, weekly) or gemini-2.5-pro (~$0.15, quarterly)
  • Single prompt returns strict JSON with exactly 42 attributes, no prose

06

Multi-Variant Generation

generation
  • Orchestrator: ScriptSage → queue-app → { vidgen-engine · imagegen-engine }
  • Thompson Sampling selects each attribute arm at generation time
  • Low-data categories (<50 videos) bootstrap from blended Bayesian priors — 60% own + 40% related
  • Diversity constraint: <30% variants share a hook type; 10+ variants per duration bucket
  • SHA-256 script-text dedup prevents near-duplicate variants

8 primary bandit attributes:

hook_type · 8 arms duration_bucket · 5 cta_placement · 3 cta_type · 5 audio_type · 4 demo_type · 5 emotion_arc · 5 caption_style · 3
07

DNA Compliance Gate

gate

Post-generation Gemini re-analysis across 7 weighted dimensions. Composite ≥ 0.70 is required before distribution. Longer videos are scored in 15-second chunks to surface weak segments.

Distribution

09

Performance Ingestion

learning
  • TikTok Business API pull every 6 hours: impressions, plays, revenue, conversions, spend
  • Derive gpm = revenue / plays × 1000
  • Minimum signal floor: 1,000 views per video before update eligibility
  • Append-only writes to dna_performance_feedback with bandit_updated=false
10

Bandit Update + Drift Detection

learning
  • Reward = 1 if video gpm ≥ P50 baseline for its category, else 0
  • Per attribute value, update Beta: α += reward, β += (1 − reward)
  • Persist dna_bandits(attribute, category, arm_value, α, β)
  • Weekly drift: Jensen-Shannon divergence vs baseline; alert to Slack #dna-alerts if ≥ 0.10
  • Auto-reset baseline after 3 consecutive weeks of same-direction drift — priors rebuild from fresh profile
