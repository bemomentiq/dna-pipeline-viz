18 · Self-learning compounder

The generator itself gets better every week, not just the bandits

A bandit can only pick from what the generator produces. For continuous improvement to compound, the generator must absorb the bandit’s winning distribution and re-emit improved variants. Five complementary mechanisms — VideoDPO, RLAIF, ORPO, GAD, and a 3-phase curriculum — turn MomentIQ’s pipeline from template-configured to self-aligning.

5.1

VideoDPO / Flow-DPO

weekly · TRL

Every week, winning generations become chosen and their nearest rejected siblings become rejected. The video diffusion model is fine-tuned directly on this preference dataset using the DPO loss with KL anchoring against a frozen reference. 

config = DPOConfig(beta=0.1, loss_type="sigmoid", lr=1e-5)
trainer = DPOTrainer(
  model=video_gen,
  ref_model=frozen_ref,      # KL anchor prevents over-opt
  train_dataset=pref_pairs,  # (prompt, chosen, rejected)
)
trainer.train()  # weekly cadence

5.2

GAD · indistinguishability

quarterly · Nash game

A discriminator learns to tell MomentIQ output from top-creator output. The generator learns to fool it. Unlike static reward models, the discriminator co-evolves — preventing reward hacking. Global optimum = discriminator accuracy approaches 0.5 (chance) = indistinguishable by construction. 

# Adversarial phase
for batch in loop:
  gen = G.generate(batch.prompts)
  # generator reward = how well it fools D
  G_loss = -mean(D(gen))
  G_opt.step(G_loss)
  # update D on fresh G output
  D_loss = BCE(D(real),1) + BCE(D(gen),0)
  D_opt.step(D_loss)

5.3

RLAIF · constitutional

nightly · annotation-free

Replace human annotation with a written content constitution (hook-in-3s, product-visible-in-8s, social-proof, CTA-clarity, trust-signal). A frontier VLM (Gemini 2.5 Pro) auto-generates preference pairs using the constitution as rubric. Preference data scales as fast as compute.

5.4

ORPO · anti-hacking

always-on · χ² reg

Occupancy-regularized policy optimization: add a χ² divergence penalty between the current policy’s state-occupancy measure and the reference policy’s. Provides a provable lower bound on true-reward improvement — the generator cannot drift into high-proxy-reward but low-GMV regions. 

# Loss with occupancy regularization
loss = -reward(pi_out) + beta * chi2_div(
  d_policy(s),    # occupancy measure
  d_ref(s)
)

Phase 2 · SynthesizeRLAIF + VideoDPO
