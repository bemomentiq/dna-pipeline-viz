If something breaks — this card first

One-page operator guide. The five most common incidents, the three most common diagnostic queries, and the emergency pause.

Emergency · pause writes
FF dna_bandit_writes = false
FF dna_accept_new   = false
SELECT * FROM dna_performance_feedback
 WHERE bandit_updated = false;
Stops any further learning; in-flight jobs drain normally.
Symptom · Drift alert fired
  1. Check #dna-alerts for attribute + divergence.
  2. Query dna_profiles for last 4 weeks vs baseline.
  3. If 3 consecutive weeks → auto-reset fires.
  4. Manual reset: Learning owner approval only.
Symptom · Compliance < 0.70
  1. Inspect failing dimension in dna_compliance.
  2. Most common: caption burn-in + hook mismatch.
  3. Check if Gemini prompt version changed.
  4. Rerun scorer with --verbose.
Symptom · Feedback lag > 12h
  1. Check TikTok Business API token (rotates every 24h).
  2. Retry last 3 ingest windows idempotently.
  3. If still failing → page Partners DRI.
Symptom · yt-dlp 403
  1. Rotate cookies via scripts/refresh-cookies.sh.
  2. Fallback to Kalodata preview URLs.
  3. Replay the failed video IDs from today's run.
Symptom · GMV lift < 0% for 2 weeks
  1. Inspect per-category lift; identify the loser.
  2. Check if a new DNA config over-indexed on one arm.
  3. Ramp back to previous step (see Rollout).
  4. Freeze bandit writes while investigating.
Three queries to know
-- Which arm is winning for a category?
SELECT attribute, arm_value, alpha, beta,
       alpha::float / (alpha + beta) AS mean
FROM dna_bandits
WHERE category = 'vacuums'
ORDER BY mean DESC;

-- Which feedback rows still unprocessed?
SELECT tiktok_video_id, window_end, views
FROM dna_performance_feedback
WHERE bandit_updated = false
ORDER BY window_end DESC;

-- 7-day compliance trend
SELECT DATE_TRUNC('day', analyzed_at) d, AVG(composite) c
FROM dna_compliance
GROUP BY 1 ORDER BY 1 DESC LIMIT 7;
